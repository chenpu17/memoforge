# ForgeNerve 应用启动页文案层级

> 日期: 2026-04-09
> 状态: 提案
> 适用范围: 桌面应用欢迎页 / 冷启动首屏

---

## 1. 目标

启动页的目标不是“告诉用户这是个文件工具”，而是：

1. 让用户知道 ForgeNerve 是面向开发者与 Agent 的知识工作台
2. 让用户快速理解三条入口的差异
3. 让首次打开的感受更像产品，而不是工程工具

---

## 2. 推荐文案层级

### 第一层：品牌名

`欢迎使用 ForgeNerve`

要求：

- 字号最大
- 一眼可识别

### 第二层：产品定位

`面向 AI Agent 的开发者知识操作系统`

要求：

- 作为一句话定位
- 帮助新用户快速建立心智

### 第三层：价值说明

`在一个 Git 原生工作台中管理知识、连接 Agent，并安全审阅 AI 生成的变更。`

要求：

- 1 行即可
- 不要堆术语

### 第四层：信任标签

- `Local-first`
- `Git-native`
- `MCP-ready`

要求：

- 用小标签展示
- 作为视觉辅助，不要抢主文案

---

## 3. 三条入口推荐文案

### 新建知识库

- 标题：`新建知识库`
- 描述：`从模板或空白工作区开始，快速搭建你的知识系统`

### 导入已有目录

- 标题：`导入已有目录`
- 描述：`打开现有 Markdown 文件夹，把已有笔记迁移到 ForgeNerve`

### Clone Git 仓库

- 标题：`Clone Git 仓库`
- 描述：`从远程仓库拉取知识库，与团队共享同一套知识资产`

---

## 4. 只读模式文案

### 提示文案

`当前为只读模式：你仍可打开已有知识库，但不能新建或克隆。`

要求：

- 明确“还能做什么”
- 不只强调限制
