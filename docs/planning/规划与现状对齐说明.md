## 1. 文档目的

这份文档只做一件事:

- 把“规划文档里的目标状态”和“当前已审计范围内的真实实现状态”显式对齐

这样做的原因很简单:

- MemoForge 已经进入第二轮产品增强阶段
- 仓库中同时存在“阶段回顾文档”和“下一阶段规划文档”
- 如果不把差异写清楚，后续排期、验收、测试都会基于错误前提推进

---

## 2. 当前结论

截至 2026-04-08，MemoForge 的真实状态应理解为:

1. 核心知识库、MCP、桌面端基础闭环已经可用。
2. Agent 协作增强和 Tauri 桌面增强两条线，目前仍处于方案与排期阶段，不应误判为“已开发完成”。
3. 部分旧文档里的“已完成”表述，描述的是阶段目标或底层能力到位，不代表当前 GUI 和完整产品流程已经到位。

---

## 3. 已确认的关键差异

### 3.1 Clone 冷启动: core 有基础，桌面流未完成

现状证据:

- `crates/memoforge-tauri/src/main.rs` 中 `init_kb_cmd` 对 `mode="clone"` 直接返回 `Clone not supported in this command`
- 前端 `frontend/src/services/tauri.ts` 也明确在 HTTP 模式下拒绝 clone 初始化

结论:

- 不能把“桌面端支持 clone 首次使用流程”视为当前已交付能力
- 该能力仍属于 `Tauri桌面应用增强开发计划` 中 Sprint A 的待开发项

### 3.2 Welcome Flow / Template Picker 仍未落地

现状证据:

- 前端组件树中尚无 `WelcomeFlow`、`TemplatePicker`
- 当前入口仍以知识库切换、打开目录、初始化目录等工程入口为主

结论:

- 冷启动体验仍偏工程化
- “首次使用成功率提升”仍是有效问题，不是已解决问题

### 3.3 Workspace Dashboard / Knowledge Health 尚未实现

现状证据:

- 前端组件中尚无 `WorkspaceDashboard`、`KnowledgeHealthPanel`、`RecentActivityPanel`
- 当前主界面仍以浏览、编辑、搜索、图谱、Git 面板为主

结论:

- MemoForge 当前是“可用桌面知识工具”
- 还不是“以任务驱动的知识工作台”

### 3.4 Agent Draft 工作流仍未落地

现状证据:

- 当前仓库中尚无 `AgentDraftPanel`、`DraftPreviewModal`
- 当前 MCP 仍以已有工具集为主，未看到 `start_draft` / `preview_draft` / `commit_draft` 的完整落地

结论:

- Agent 友好写入目前仍然主要依赖整篇内容写入和整篇替换
- Draft 机制和 section 级写入仍属于下一阶段重点

### 3.5 设置页已有帮助增强，但还不是完整偏好中心

现状证据:

- `frontend/src/components/SettingsModal.tsx` 已补充 MCP 快速配置和诊断信息
- 但当前设置中心仍以诊断、连接信息和帮助为主

结论:

- 设置页增强已经开始
- 但“完整偏好中心”仍属于桌面端下阶段工作

## 4. 对排期与验收的影响

从现在开始，建议统一采用以下判断口径:

### 4.1 “已完成”

只能用于以下情况:

- 代码已存在
- 主路径可运行
- 有对应测试或可操作验证路径

### 4.2 “已具备基础能力”

用于以下情况:

- 底层 core 或 API 能力已存在
- 但 GUI、工作流或产品化体验尚未完成

### 4.3 “规划中 / 待开发”

用于以下情况:

- 文档已经定义
- 代码尚未具备完整用户可用形态

---

## 5. 建议执行方式

为了避免后续继续出现“文档先于实现过多”的问题，建议按下面的顺序推进:

1. 先把 Agent 线 Sprint 1 和 Tauri 线 Sprint A 拆到 issue 粒度。
2. 每个 Sprint 只承诺一个可验收主路径，不同时宣称多个产品化能力完成。
3. README、开发计划文档、发布说明三处统一维护状态口径。
4. 每轮发布前补一份“规划与现状差异复核”。

---

## 6. 当前推荐口径

可以对外或对团队内部统一表述为:

- MemoForge 当前已经具备可用的桌面知识库、MCP 接入、Git 集成和基础知识管理能力。
- 下一轮版本重点不是“从 0 到 1”，而是把 Agent 协作写入和桌面工作台体验从可用提升到稳定、顺手、可确认。
- Agent Draft 流、桌面欢迎流、Clone GUI、Workspace Dashboard、Knowledge Health、Agent Draft Panel 当前都仍在规划与拆解阶段。
