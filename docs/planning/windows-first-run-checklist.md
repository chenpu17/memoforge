# Windows First-Run Checklist

## 目标

覆盖 Windows 首次使用、多知识库切换、异常恢复、日志反馈四条最容易出现“没响应”错觉的链路。

## 用例

1. 首次打开空目录
   期望：点击“打开或初始化知识库”后自动创建 `.memoforge`，主界面正常进入。

2. 打开已初始化目录
   期望：正常进入主界面，不重复初始化，不破坏已有 `.memoforge/config.yaml`。

3. 打开非空未初始化目录
   期望：页面内显示明确错误，说明该目录不是知识库，并提示空目录才会自动初始化。

4. 自动打开最近知识库
   期望：重启应用后自动进入最近使用的知识库。

5. 最近知识库失效
   操作：删除或移动最近知识库目录后重启应用。
   期望：停留在初始化页，并展示“自动打开失败”的具体原因。

6. 多知识库切换
   期望：从知识库 A 切到 B 后，标题、树、文档内容、Git 状态都刷新为 B。

7. 注销历史知识库
   期望：仅移除注册记录，不删除文件目录。

8. 日志目录可见
   期望：设置页能显示日志目录、日志文件、当前知识库、最近日志。

9. 日志目录可打开
   期望：点击“打开日志目录”能打开系统文件管理器。

10. 空目录初始化失败回报
    操作：选择一个无写权限目录。
    期望：页面内显示错误，日志里有 `init_kb` 失败记录。

## 建议记录

- Windows 版本
- 安装包版本
- 目录路径是否包含中文、空格、OneDrive/网络盘
- 设置页“最近日志”内容
- 是否出现系统文件夹选择器异常
