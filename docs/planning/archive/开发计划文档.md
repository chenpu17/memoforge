#### Sprint 9 实现详情

**9.1 `summary_stale` 检测：**
- `Frontmatter` 新增 `summary_hash: Option<String>` 字段
- 内容更新时保持 hash 不变，摘要更新时重新计算 hash
- 前端 `MetadataPanel` 显示过期警告（黄色背景）

**9.2 自动分页：**
- `list_knowledge` 返回 `PaginatedKnowledge` 结构体
- 包含 `total`, `limit`, `offset`, `has_more` 字段
- 前端实现"加载更多"按钮

**9.3 HTTP Server（Axum）：**
- 新建 `memoforge-http` crate
- 实现完整 REST API：
  - `GET /api/status` - 知识库状态
  - `GET /api/knowledge` - 列表（分页、标签过滤）
  - `GET /api/knowledge/:id` - 获取详情
  - `GET /api/categories` - 分类列表
  - `GET /api/tags` - 标签列表
  - `POST /api/knowledge` - 创建（需认证）
  - `PUT /api/knowledge/:id` - 更新（需认证）
  - `DELETE /api/knowledge/:id` - 删除（需认证）

**9.4 Web 安全模型：**
- 路径遍历防护（`validate_kb_path`）
- Bearer Token 认证
- CORS 配置（默认禁用）
- Rate Limiting（防内存泄漏优化）
- 只读模式强制

**9.6 Web 端只读浏览：**
- `isWebMode` 检测（`!isTauri`）
- `ReadOnlyBanner` 组件显示只读提示
- 隐藏编辑/新建按钮
- 禁用元数据编辑

#### Phase 3 代码审核与修复

完成两轮代码审核，修复以下问题：
1. ✅ 路径遍历漏洞 - 添加 `validate_kb_path` 函数
2. ✅ CORS 配置重复 - 删除重复文件

## 7. 关键技术风险与应对

| 风险 | 影响 | 应对措施 |
|------|------|---------|
| ~~Milkdown 编辑器集成复杂度~~ | ~~可能影响 Sprint 5 进度~~ | ✅ 已用 CodeMirror + react-markdown 替代 |
| git2-rs 在 Windows 上的编译问题 | 跨平台兼容 | 优先支持 macOS/Linux，Windows 后续测试修复 |
| MCP 协议规范变化 | tool 注册方式可能调整 | 关注 MCP spec 更新，保持 tool 层与核心逻辑解耦 |
| 多进程文件锁在 NFS/网络文件系统不可靠 | 网络挂载的知识库可能锁失败 | 文档说明仅支持本地文件系统 |
