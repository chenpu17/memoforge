# 前端实时状态发布测试指南

## 测试环境
- Tauri 桌面应用
- 知识库已初始化
- 后端 MCP Server 运行中（可选，用于验证状态接收）

## 测试步骤

### 1. 启动应用测试
```bash
cd /Users/chenpu/workspace/claude-code/知识库
./start.sh
```

### 2. 验证知识库状态发布

**测试点**：应用启动时是否正确发布知识库状态

**步骤**：
1. 启动应用
2. 观察控制台日志（无错误）
3. 检查后端是否收到知识库状态

**预期结果**：
```
[EditorState] 发布知识库状态成功 (或无错误日志)
```

### 3. 验证知识点选择状态

**测试点**：选择知识点时是否正确发布状态

**步骤**：
1. 在侧边栏点击某个知识点
2. 切换到"编辑"模式
3. 观察控制台日志

**预期结果**：
```
[EditorState] selectKnowledge called: {path: "...", title: "...", category: "..."}
```

### 4. 验证选区实时更新

**测试点**：选择文本时是否实时发布选区状态（100ms 防抖）

**步骤**：
1. 在编辑器中选择一段文本
2. 立即观察控制台（应该在 100ms 后看到日志）
3. 取消选择，观察是否立即清除状态

**预期结果**：
```
选择文本后 100ms: [EditorState] updateSelection: {startLine: X, endLine: Y}
取消选择时: [EditorState] clearSelection called
```

### 5. 验证知识点切换

**测试点**：切换知识点时是否正确清理和设置状态

**步骤**：
1. 选择知识点 A
2. 选择文本（产生选区）
3. 切换到知识点 B
4. 观察控制台日志

**预期结果**：
```
选择 A: selectKnowledge(A)
选择文本: updateSelection(line X, line Y)
切换 B: clearKnowledge() -> selectKnowledge(B)
```

### 6. 验证知识库切换

**测试点**：切换知识库时是否正确发布状态

**步骤**：
1. 打开知识库切换器
2. 切换到另一个知识库
3. 观察控制台日志

**预期结果**：
```
切换知识库: setKb(newPath, newName, count)
```

### 7. 验证防抖机制

**测试点**：快速连续选择是否正确防抖

**步骤**：
1. 快速连续选择不同文本（间隔 < 100ms）
2. 观察实际调用次数

**预期结果**：
```
应该只看到最后一次选区的 updateSelection 调用
```

### 8. 验证组件卸载

**测试点**：组件卸载时是否正确清理定时器

**步骤**：
1. 在编辑器中选择文本
2. 在防抖期间（100ms 内）关闭应用或切换知识点
3. 观察是否有内存泄漏或错误

**预期结果**：
```
无错误日志，定时器被正确清理
```

## 错误场景测试

**预期**：
```
[EditorState] 发布XXX状态失败: Error: ...
应用继续运行，不崩溃
```

### 2. HTTP 模式
**环境**：浏览器中运行（非 Tauri）

**预期**：
```
无状态发布调用，应用正常运行
```

## 已知问题和解决方案

### 问题 1: 防抖期间切换知识点
**现象**：切换知识点后仍收到旧选区的状态更新

**解决**：在切换时调用 `clearSelection()` 立即清除定时器

### 问题 2: 知识库数量更新频繁
**现象**：知识库列表变化时多次调用 `setKb`

**解决**：已添加 `useEffect` 依赖优化
