## 2. AI 协作机制实现

### 2.1 设计概述（v2.0 新架构）

MemoForge 的核心差异化特性是让 AI Agent 能够感知用户的工作上下文。**v2.0 采用 Pencil 风格架构**：

- **GUI 内嵌 MCP Server** - 桌面应用直接提供 MCP SSE 服务
- **内存直接访问** - 无需文件同步，实时获取编辑器状态
- **零延迟感知** - 选区变化立即对 AI Agent 可见
- **单进程架构** - 部署简单，无状态同步问题

### 2.2 架构对比

| 特性 | v1.x (文件同步) | v2.0 (混合同步) |
|------|----------------|-----------------|
| 状态同步 | `~/.memoforge/editor_state.yaml` | SSE 直读内存状态，CLI follow/bound 仍兼容 `editor_state.yaml` |
| 延迟 | 300ms 节流 + 文件 I/O | 实时（<1ms） |
| 可靠性 | 文件锁竞争、竞态条件 | 无并发问题 |
| 部署 | 两个进程 | 单进程 |
| MCP 协议 | stdio | SSE (Server-Sent Events) |

### 2.4 内存状态管理

```rust
// crates/memoforge-tauri/src/memory_state.rs

use std::sync::{Arc, Mutex};
use serde::{Deserialize, Serialize};
use chrono::{DateTime, Utc};
use std::path::PathBuf;

/// 内存中的编辑器状态
#[derive(Debug, Clone, Default)]
pub struct EditorState {
    /// 当前知识库
    pub current_kb: Option<CurrentKb>,
    /// 当前选中的知识点
    pub current_knowledge: Option<CurrentKnowledge>,
    /// 文本选择范围
    pub selection: Option<Selection>,
    /// 桌面应用是否聚焦
    pub focused: bool,
    /// 最后更新时间
    pub updated_at: DateTime<Utc>,
}

#[derive(Debug, Clone, Serialize, Deserialize)]
pub struct CurrentKb {
    pub path: PathBuf,
    pub name: String,
    pub knowledge_count: usize,
}

#[derive(Debug, Clone, Serialize, Deserialize)]
pub struct CurrentKnowledge {
    pub path: String,           // 相对路径
    pub title: String,
    pub category: Option<String>,
}

#[derive(Debug, Clone, Serialize, Deserialize)]
pub struct Selection {
    pub start_line: usize,
    pub end_line: usize,
    pub has_text: bool,
    pub text_length: usize,
    /// 实际选中文本（仅当用户显式开启时才返回）
    #[serde(skip_serializing_if = "Option::is_none")]
    pub selected_text: Option<String>,
}

/// 全局状态管理器
pub struct StateManager {
    state: Arc<Mutex<EditorState>>,
}

impl StateManager {
    pub fn new() -> Self {
        Self {
            state: Arc::new(Mutex::new(EditorState::default())),
        }
    }

    /// 获取当前状态（MCP 工具调用）
    pub fn get_state(&self) -> EditorState {
        self.state.lock().unwrap().clone()
    }

    /// 更新知识库（GUI 调用）
    pub fn set_kb(&self, path: PathBuf, name: String, count: usize) {
        let mut state = self.state.lock().unwrap();
        state.current_kb = Some(CurrentKb { path, name, knowledge_count: count });
        state.updated_at = Utc::now();
    }

    /// 更新当前知识点（GUI 调用）
    pub fn set_knowledge(&self, path: String, title: String, category: Option<String>) {
        let mut state = self.state.lock().unwrap();
        state.current_knowledge = Some(CurrentKnowledge { path, title, category });
        state.updated_at = Utc::now();
    }

    /// 更新选区（GUI 调用，实时）
    pub fn set_selection(&self, start_line: usize, end_line: usize, text: Option<String>) {
        let mut state = self.state.lock().unwrap();
        let has_text = text.is_some();
        let text_length = text.as_ref().map(|t| t.len()).unwrap_or(0);

        state.selection = Some(Selection {
            start_line,
            end_line,
            has_text,
            text_length,
            selected_text: text, // 根据 privacy 配置过滤
        });
        state.updated_at = Utc::now();
    }

    /// 清除知识点选择
    pub fn clear_knowledge(&self) {
        let mut state = self.state.lock().unwrap();
        state.current_knowledge = None;
        state.selection = None;
        state.updated_at = Utc::now();
    }
}
```

### 2.6 SSE MCP Server 实现

```rust
// 概念性示例：对应当前实现的 crates/memoforge-mcp/src/sse.rs

use axum::{
    routing::{get, post},
    Router,
    response::{sse::Event, Sse},
    extract::State,
    http::StatusCode,
    body::Body,
};
use std::sync::Arc;
use tokio::sync::mpsc;
use futures::stream::{Stream, StreamExt};
use serde_json::{json, Value};

/// MCP Server 状态
pub struct McpServerState {
    pub editor_state: Arc<StateManager>,
    pub kb_path: Arc<Mutex<Option<PathBuf>>>,
}

/// 启动 SSE MCP Server
pub async fn start_mcp_server(state: Arc<McpServerState>, port: u16) {
    let app = Router::new()
        // MCP 协议端点
        .route("/mcp", post(handle_mcp_request))
        // SSE 端点（用于长时间运行的请求）
        .route("/mcp/sse", get(handle_mcp_sse))
        // 健康检查
        .route("/health", get(|| async { "OK" }))
        .with_state(state);

    let addr = format!("127.0.0.1:{}", port);
    let listener = tokio::net::TcpListener::bind(&addr).await.unwrap();

    axum::serve(listener, app).await.unwrap();
}

/// 处理 MCP 请求
async fn handle_mcp_request(
    State(server_state): State<Arc<McpServerState>>,
    body: String,
) -> Result<String, StatusCode> {
    let request: JsonRpcRequest = serde_json::from_str(&body)
        .map_err(|_| StatusCode::BAD_REQUEST)?;

    let response = match request.method.as_str() {
        "initialize" => handle_initialize(),
        "tools/list" => handle_tools_list(),
        "tools/call" => handle_tools_call(request.params, &server_state),
        _ => JsonRpcResponse::error(request.id, -32601, "Method not found"),
    };

    Ok(serde_json::to_string(&response).unwrap())
}

/// 初始化响应
fn handle_initialize() -> JsonRpcResponse {
    JsonRpcResponse::success(json!({
        "protocolVersion": "2024-11-05",
        "capabilities": {
            "tools": {}
        },
        "serverInfo": {
            "name": "memoforge",
            "version": env!("CARGO_PKG_VERSION")
        }
    }))
}

/// 工具列表
fn handle_tools_list() -> JsonRpcResponse {
    JsonRpcResponse::success(json!({
        "tools": [
            {
                "name": "get_editor_state",
                "description": "获取用户当前的编辑器状态。包括当前打开的知识库、选中的知识点、文本选择范围等。调用此工具可以了解用户正在关注的内容，无需用户每次都说明上下文。",
                "inputSchema": { "type": "object", "properties": {} }
            },
            // ... 其他 25 个工具
        ]
    }))
}

/// 处理工具调用
fn handle_tools_call(
    params: Option<Value>,
    server_state: &McpServerState,
) -> JsonRpcResponse {
    let params = match params {
        Some(p) if p.is_object() => p,
        _ => return JsonRpcResponse::error(None, -32602, "Invalid params"),
    };

    let tool_name = params["name"].as_str().unwrap_or("");

    match tool_name {
        "get_editor_state" => {
            let state = server_state.editor_state.get_state();
            JsonRpcResponse::success(json!({
                "current_kb": state.current_kb,
                "current_knowledge": state.current_knowledge,
                "selection": state.selection,
                "focused": state.focused,
                "updated_at": state.updated_at
            }))
        }
        "list_knowledge" => {
            // 从内存状态获取当前 KB，执行操作
            let kb_path = server_state.editor_state.get_state().current_kb
                .map(|kb| kb.path);
            // ... 调用 memoforge-core 执行操作
            todo!()
        }
        _ => JsonRpcResponse::error(None, -32601, "Unknown tool"),
    }
}

```typescript
// frontend/src/hooks/useEditorStatePublisher.ts

import { useEffect, useCallback, useRef } from 'react'
import { invoke } from '@tauri-apps/api/core'

export function useEditorStatePublisher() {
  const debounceTimer = useRef<number | null>(null)

  // 选择知识点时发布状态
  const selectKnowledge = useCallback(async (
    path: string,
    title: string,
    category?: string
  ) => {
    await invoke('select_knowledge_cmd', { path, title, category })
  }, [])

  // 更新选区（实时，100ms 防抖）
  const updateSelection = useCallback((
    startLine: number,
    endLine: number,
    text?: string
  ) => {
    if (debounceTimer.current) {
      clearTimeout(debounceTimer.current)
    }

    debounceTimer.current = setTimeout(() => {
      invoke('update_selection_cmd', { startLine, endLine, text })
    }, 100)
  }, [])

  // 清除选择
  const clearKnowledge = useCallback(async () => {
    await invoke('clear_knowledge_cmd')
  }, [])

  return { selectKnowledge, updateSelection, clearKnowledge }
}

export const Editor: React.FC<EditorProps> = ({ value, knowledgePath }) => {
  const { updateSelection } = useEditorStatePublisher()

  const handleSelectionChange = useCallback((selection: SelectionRange) => {
    if (!knowledgePath) return

    const from = selection.main.from
    const to = selection.main.to

    if (from === to) return // 光标位置，无选区

    // 计算行号（1-based）
    const lines = value.substring(0, from).split('\n')
    const startLine = lines.length
    const endLines = value.substring(0, to).split('\n')
    const endLine = endLines.length

    // 提取选中文本
    const selectedText = value.substring(from, to)

    // 发布状态（100ms 防抖）
    updateSelection(startLine, endLine, selectedText)
  }, [value, knowledgePath, updateSelection])

  return (
    <CodeMirror
      value={value}
      onSelectionChange={handleSelectionChange}
      // ...
    />
  )
}

#### 2.5.1 CLI 定义与模式

当前 MCP 对外接入有三种模式：桌面端内嵌 SSE、CLI `follow`、CLI `bound`。其中 `--mode` 参数仅作用于 CLI fallback：

```rust
// crates/memoforge-mcp/src/cli.rs

#[derive(Parser)]
#[command(name = "memoforge")]
enum Cli {
    Serve {
        /// 运行模式：follow(跟随当前编辑器) 或 bound(绑定指定知识库)
        /// 默认为 follow，此时每次调用都读取当前 GUI 的知识库
        #[arg(long, default_value = "follow", value_parser = ["follow", "bound"])]
        mode: String,

        /// 知识库路径（bound 模式必填，follow 模式忽略）
        #[arg(long, required_if_eq("mode", "bound"))]
        knowledge_path: Option<PathBuf>,

        /// [follow 模式] 允许在状态无效时回退到最近使用的知识库（仅只读操作）
        /// 默认 false：状态无效时所有操作都返回 NOT_INITIALIZED 错误
        /// 设置为 true：只读操作（list/get/grep/search）可使用最近知识库
        #[arg(long, default_value = "false")]
        allow_stale_kb: bool,

        /// Agent 名称（用于状态显示）
        #[arg(long, default_value = "unknown")]
        agent_name: String,

        // ... 其他参数
    },
}
```

**CLI 两种模式的行为差异：**

| 操作 | 绑定模式 (bound) | 跟随模式 (follow) | 跟随模式 + `--allow-stale-kb` |
|------|------------------|-------------------|------------------------------|
| 启动时验证 | 验证 KB 路径有效 | 不验证 | 不验证 |
| 获取 KB 路径 | 从启动参数获取 | 每次调用从共享状态获取 | 同左 |
| 状态有效时写入 | ✅ 直接写入 | ✅ 验证后写入 | ✅ 验证后写入 |
| 状态无效时写入 | ✅ 直接写入 | ❌ `NOT_INITIALIZED` | ❌ `NOT_INITIALIZED` |
| 状态无效时只读 | ✅ 直接读取 | ❌ `NOT_INITIALIZED` | ⚠️ 使用最近 KB |

```rust
// 启动时解析运行模式和路径
fn resolve_mode_and_path(cli: &Cli) -> Result<(EditorMode, Option<PathBuf>), MemoError> {
    let mode = match cli.mode.as_str() {
        "follow" => EditorMode::Follow { allow_stale_kb: cli.allow_stale_kb },
        "bound" => EditorMode::Bound,
        _ => return Err(MemoError::invalid_argument("mode", "必须是 follow 或 bound")),
    };

    match mode {
        EditorMode::Bound => {
            // 绑定模式必须有路径
            let path = cli.knowledge_path.clone()
                .ok_or_else(|| MemoError::invalid_argument("knowledge-path", "绑定模式必须指定"))?;
            Ok((EditorMode::Bound, Some(path)))
        }
        EditorMode::Follow { allow_stale_kb } => {
            // 跟随模式：路径从共享状态获取，启动时不验证
            Ok((EditorMode::Follow { allow_stale_kb }, None))
        }
    }
}

#### 2.6.2 编辑器选区监听

CodeMirror 编辑器需要监听用户选区变化，实时同步到共享状态：

```typescript
// frontend/src/components/Editor.tsx

import CodeMirror, { SelectionRange } from '@uiw/react-codemirror'
import { useCallback, useRef, useEffect } from 'react'

interface EditorProps {
  value: string
  onChange: (value: string) => void
  mode: 'read' | 'edit'
  knowledgePath?: string  // 当前知识点路径
  onSelectionChange?: (selection: SelectionInfo | null) => void
}

interface SelectionInfo {
  start_line: number
  end_line: number
  selected_text: string
}

export const Editor: React.FC<EditorProps> = ({
  value,
  onChange,
  mode,
  knowledgePath,
  onSelectionChange
}) => {
  const lastSelectionRef = useRef<SelectionInfo | null>(null)
  const debounceTimerRef = useRef<NodeJS.Timeout | null>(null)

  // 处理选区变化（带防抖，避免频繁写入）
  const handleSelectionChange = useCallback((selection: SelectionRange) => {
    if (!onSelectionChange || !knowledgePath) return

    // 清除之前的定时器
    if (debounceTimerRef.current) {
      clearTimeout(debounceTimerRef.current)
    }

    // 防抖 300ms
    debounceTimerRef.current = setTimeout(() => {
      const from = selection.main.from
      const to = selection.main.to

      // 没有选中文本（光标位置）
      if (from === to) {
        if (lastSelectionRef.current !== null) {
          lastSelectionRef.current = null
          onSelectionChange(null)
        }
        return
      }

      // 计算行号（1-based）
      const lines = value.substring(0, from).split('\n')
      const startLine = lines.length
      const endLines = value.substring(0, to).split('\n')
      const endLine = endLines.length

      // 提取选中文本
      const selectedText = value.substring(from, to)

      const selectionInfo: SelectionInfo = {
        start_line: startLine,
        end_line: endLine,
        selected_text: selectedText,
      }

      // 只有变化时才更新
      if (
        lastSelectionRef.current?.start_line !== selectionInfo.start_line ||
        lastSelectionRef.current?.end_line !== selectionInfo.end_line ||
        lastSelectionRef.current?.selected_text !== selectionInfo.selected_text
      ) {
        lastSelectionRef.current = selectionInfo
        onSelectionChange(selectionInfo)
      }
    }, 300)
  }, [value, knowledgePath, onSelectionChange])

  // 组件卸载时清理选区
  useEffect(() => {
    return () => {
      if (debounceTimerRef.current) {
        clearTimeout(debounceTimerRef.current)
      }
    }
  }, [])

  // 编辑模式
  if (mode === 'edit') {
    return (
      <div className="h-full w-full">
        <CodeMirror
          value={value}
          height="100%"
          extensions={[markdown(), EditorView.updateListener.of((update) => {
            if (update.selectionSet) {
              handleSelectionChange(update.state.selection)
            }
          })]}
          onChange={onChange}
          // ... 其他配置
        />
      </div>
    )
  }

  // 阅读模式 - 监听 DOM 选区
  return (
    <div
      className="h-full w-full overflow-auto"
      onMouseUp={handleReadModeSelection}
      onKeyUp={handleReadModeSelection}
    >
      <ReactMarkdown ... />
    </div>
  )

  function handleReadModeSelection() {
    const selection = window.getSelection()
    if (!selection || selection.isCollapsed) {
      onSelectionChange?.(null)
      return
    }

    const text = selection.toString()
    if (!text.trim()) {
      onSelectionChange?.(null)
      return
    }

    // 阅读模式下只能获取选中文本，无法获取精确行号
    onSelectionChange?.({
      start_line: 0,
      end_line: 0,
      selected_text: text,
    })
  }
}

    //
    // 1. git add -u   （暂存已追踪文件的修改和删除，覆盖 delete_knowledge / move_knowledge 的旧路径）
    // 2. git add .     （暂存新文件，.gitignore 自动排除运行时文件）
    //    等价于 git add -A，但因为有 .gitignore 保护，运行时文件不会被暂存
    // 3. git status --porcelain 检查是否有待提交内容
    //    如无变更 → 返回 "nothing to commit"（非错误）
    // 4. git commit -m message
    //
    // 安全兜底：在 git add 后、commit 前，检查 staged files 中是否包含
    // .memoforge/ 下除 config.yaml 和 .gitignore 外的文件，如有则 unstage 并警告
}

#[derive(Serialize, Deserialize, Clone)]
pub struct Frontmatter {
    pub id: String,
    pub title: String,
    #[serde(default)]
    pub tags: Vec<String>,
    #[serde(default)]
    pub category: Option<String>,
    #[serde(default)]
    pub summary: Option<String>,
    #[serde(default)]
    pub summary_hash: Option<String>,
    pub created_at: DateTime<Utc>,
    pub updated_at: DateTime<Utc>,
}

/// 三态 Patch 值：区分"不更新"、"设为新值"、"清空为 null"
/// JSON 映射：字段缺失 = Absent，字段有值 = Set(value)，字段为 null = Clear
///   { "summary": "new text" }  → summary = Set("new text")
///   { "summary": null }        → summary = Clear
///   { }                        → summary = Absent（不动）
#[derive(Clone, Default)]
pub enum PatchValue<T> {
    #[default]
    Absent,       // 字段未出现 → 保持原值
    Set(T),       // 字段有值 → 更新为此值
    Clear,        // 字段为 null → 清空（对 Option 字段设为 None，对 Vec 字段设为空）
}

/// Patch 类型：用于 update/update_metadata 的增量更新。
/// AI Agent 调用 update_metadata 时只需传想改的字段：
///   { "summary": "new summary" }       → 只更新 summary
///   { "summary": null }                → 清空 summary
///   { "tags": ["rust", "async"] }      → 替换 tags
///   { "tags": null }                   → 清空 tags
#[derive(Clone, Default)]
pub struct FrontmatterPatch {
    pub title: PatchValue<String>,
    pub tags: PatchValue<Vec<String>>,
    pub category: PatchValue<String>,
    pub summary: PatchValue<String>,
    // created_at 不可修改
    // updated_at 由系统自动设置为当前时间
    // summary_hash 在 summary 变更时自动重算
}

impl FrontmatterPatch {
    /// 将 patch 合并到现有 frontmatter
    pub fn apply_to(&self, fm: &mut Frontmatter, body: &str) -> Result<(), MemoError> {
        match &self.title {
            PatchValue::Set(v) => fm.title = v.clone(),
            PatchValue::Clear => {
                // title 必填，不允许清空 → 返回错误
                return Err(MemoError {
                    code: ErrorCode::InvalidFrontmatter,
                    message: "title is required and cannot be cleared".into(),
                    retry_after_ms: None,
                    context: None,
                });
            }
            PatchValue::Absent => {}
        }
        match &self.tags {
            PatchValue::Set(v) => fm.tags = v.clone(),
            PatchValue::Clear => fm.tags = vec![],
            PatchValue::Absent => {}
        }
        match &self.category {
            PatchValue::Set(v) => fm.category = Some(v.clone()),
            PatchValue::Clear => fm.category = None,
            PatchValue::Absent => {}
        }
        match &self.summary {
            PatchValue::Set(v) => {
                fm.summary = Some(v.clone());
                fm.summary_hash = Some(content_hash(body));
            }
            PatchValue::Clear => {
                fm.summary = None;
                fm.summary_hash = None;
            }
            PatchValue::Absent => {}
        }
        fm.updated_at = Utc::now();
        Ok(())
    }
}

        Ok(BatchLock { locks })

**move_knowledge 并发安全流程：**

```
move_knowledge(from, to, dry_run=false)
  │
  ├── 1. dry_run 阶段：扫描引用，返回影响列表（不加锁）
  │
  └── 2. 执行阶段：
      ├── a. 获取全局写锁 (lock_global_write)
      │      → 阻塞其他进程的所有写操作
      ├── b. 获取 from + to 的文件锁
      ├── c. 执行文件移动 (rename)
      ├── d. update_references: 逐个修改引用文件
      │      → 此时其他写入被全局写锁阻塞，不会出现覆盖冲突
      ├── e. 更新内存索引
      └── f. 释放全局写锁 + 文件锁
           → 其他写操作恢复
```

> **全局写锁的影响范围：** 仅在 move_knowledge 的引用更新阶段（步骤 d）持有，通常耗时 < 100ms（扫描所有 `.md` 文件正文中的 wikilink）。普通的 create/update/delete 操作在获取单文件锁前，先尝试获取 `write.lock` 的共享锁（shared lock），共享锁之间不互斥，因此不影响正常并发写入。只有 move_knowledge 的排他锁会短暂阻塞。

pub async fn auth_middleware(
    req: Request,
    next: Next,
) -> Result<Response, StatusCode> {
    let expected_token = req.extensions().get::<String>()
        .expect("token must be set");

    // 仅从 Authorization header 读取，不接受 URL query 参数
    let auth_header = req.headers()
        .get("authorization")
        .and_then(|v| v.to_str().ok());

    match auth_header {
        Some(header) if header.starts_with("Bearer ") => {
            let token = &header[7..];
            if token == expected_token.as_str() {
                Ok(next.run(req).await)
            } else {
                Err(StatusCode::UNAUTHORIZED)
            }
        }
        _ => Err(StatusCode::UNAUTHORIZED),
    }

**CORS 配置：**

```rust
use tower_http::cors::{CorsLayer, AllowOrigin};

/// CORS 策略：
/// - 同源请求不带 Origin header，浏览器不触发 CORS 检查，天然放行
/// - 跨域请求（不同 origin）默认被拒绝
/// - 仅在用户通过配置文件显式添加 allowed_origins 时才启用 CORS 层
fn build_cors(allowed_origins: &[String]) -> Option<CorsLayer> {
    if allowed_origins.is_empty() {
        // 默认：不添加 CORS 层 → 任何跨域请求都会被浏览器拦截
        // 同源请求不受影响（浏览器不发 preflight）
        None
    } else {
        // 用户配置了允许的 origins
        let origins: Vec<_> = allowed_origins.iter()
            .filter_map(|o| o.parse().ok())
            .collect();
        Some(CorsLayer::new()
            .allow_origin(AllowOrigin::list(origins))
            .allow_methods([Method::GET, Method::POST, Method::PUT, Method::DELETE])
            .allow_headers([AUTHORIZATION, CONTENT_TYPE]))
    }
}
```

> **为什么不用 `AllowOrigin::exact("null")`：** `Origin: null` 仅出现在 `file://` 协议或 sandboxed iframe 中，不是"同源"的含义。同源请求的浏览器行为是根本不带 Origin header、不触发 CORS preflight，因此不需要服务端做任何 CORS 配置。

**安全规则总结：**

| 规则 | 实现 |
|------|------|
| Token 来源 | 仅 `Authorization: Bearer <token>` header |
| URL query 传 token | 明确禁止，中间件不读取 query 参数 |
| 默认监听地址 | `127.0.0.1`（仅本机） |
| 跨域 | 默认禁止，需显式配置 |
| Token 持久化 | 写入 `.memoforge/http.token`，仅文件所有者可读（chmod 600） |

#### 2.2.4 Agent 连接状态管理

每个 MCP Server 进程启动时在 `.memoforge/agents/` 目录下创建独立的 PID 文件，支持多 Agent 连接检测。

```rust
use serde::{Deserialize, Serialize};

#[derive(Debug, Serialize, Deserialize)]
pub struct AgentInfo {
    pub pid: u32,
    pub name: String,
    pub started_at: String,
    pub kb_path: String,
}

/// 启动时写入 Agent 信息文件
fn write_agent_file(root: &Path, agent_name: &str) -> Result<(), MemoError> {
    let agents_dir = root.join(".memoforge/agents");
    std::fs::create_dir_all(&agents_dir)?;

    let pid = std::process::id();
    let info = AgentInfo {
        pid,
        name: agent_name.to_string(),
        started_at: Utc::now().to_rfc3339(),
        kb_path: root.display().to_string(),
    };

    let pid_path = agents_dir.join(format!("{}.json", pid));
    std::fs::write(&pid_path, serde_json::to_string(&info)?)?;
    Ok(())
}

/// 退出时清理 Agent 信息文件
fn cleanup_agent_file(root: &Path) {
    let pid = std::process::id();
    let _ = std::fs::remove_file(root.join(".memoforge/agents").join(format!("{}.json", pid)));
}

/// 获取所有活跃的 Agent 连接
pub fn get_active_agents(root: &Path) -> Vec<AgentInfo> {
    let agents_dir = root.join(".memoforge/agents");
    if !agents_dir.exists() {
        return vec![];
    }

    let mut agents = vec![];
    if let Ok(entries) = std::fs::read_dir(&agents_dir) {
        for entry in entries.flatten() {
            if let Ok(content) = std::fs::read_to_string(entry.path()) {
                if let Ok(info) = serde_json::from_str::<AgentInfo>(&content) {
                    // 验证进程是否存活
                    if process_alive(info.pid) {
                        agents.push(info);
                    } else {
                        // 清理已终止进程的文件
                        let _ = std::fs::remove_file(entry.path());
                    }
                }
            }
        }
    }
    agents
}

/// 检查进程是否存活
fn process_alive(pid: u32) -> bool {
    #[cfg(unix)]
    {
        // 使用 kill(pid, 0) 检查进程是否存在
        unsafe { libc::kill(pid as i32, 0) == 0 || std::io::Error::last_os_error().raw_os_error() != Some(3) }
    }
    #[cfg(windows)]
    {
        // Windows 实现
        use std::process::Command;
        Command::new("tasklist")
            .args(["/FI", &format!("PID eq {}", pid)])
            .output()
            .map(|o| String::from_utf8_lossy(&o.stdout).contains(&pid.to_string()))
            .unwrap_or(false)
    }
}

// services/api.http.ts — HTTP 实现（Web 端）
const BASE_URL = window.location.origin;

/**
 * Token 获取策略：
 * 1. 优先从 URL hash fragment 读取（首次访问时由用户粘贴到地址栏）
 * 2. 存入 sessionStorage（同一标签页会话期间复用）
 * 3. 不存入 localStorage，关闭标签页后需重新输入
 */
function getToken(): string {
    // 优先从 sessionStorage 获取
    const cached = sessionStorage.getItem('memoforge_token');
    if (cached) return cached;

    // 从 URL hash 解析: http://localhost:8080/#token=mf_xxx
    const hash = window.location.hash;
    const match = hash.match(/token=([^&]+)/);
    if (match) {
        const token = match[1];
        sessionStorage.setItem('memoforge_token', token);
        // 清除 URL 中的 token，避免浏览器历史泄漏
        window.location.hash = '';
        return token;
    }

    // 弹出输入框要求用户粘贴 token
    const input = prompt('请输入 MemoForge 访问令牌 (在终端启动日志中查看)');
    if (input) {
        sessionStorage.setItem('memoforge_token', input);
        return input;
    }

    throw new Error('未提供访问令牌');
}

export async function listKnowledge(params: ListParams): Promise<ListResult> {
    const query = new URLSearchParams(params as any).toString();
    const res = await fetch(`${BASE_URL}/api/knowledge?${query}`, {
        headers: { 'Authorization': `Bearer ${getToken()}` }
    });
    if (res.status === 401) {
        sessionStorage.removeItem('memoforge_token');
        throw new Error('令牌无效或已过期，请刷新页面重新输入');
    }
    return res.json();
}

### 6.4 冷启动路径技术设计

> 对应 PRD §3 场景6 / §5.1.6 / §9.1（Day 1 Must Have）

冷启动有三条路径，共用同一个 `KnowledgeInit` 模块：

```rust
pub enum InitMode {
    /// 从零创建新知识库（可选模板）
    New {
        path: PathBuf,
        template: Option<String>,  // "developer", "tech-notes", None(空白)
        git_remote: Option<String>, // 可选关联远程仓库
    },
    /// Clone 已有 Git 仓库
    Clone {
        remote_url: String,
        local_path: PathBuf,
    },
    /// 打开已存在的本地目录（可能是手工创建的 Markdown 文件夹）
    Open {
        path: PathBuf,
    },
}

pub struct KnowledgeInit;

impl KnowledgeInit {
    /// 统一入口：根据 InitMode 执行不同的初始化流程
    pub fn init(mode: InitMode) -> Result<KnowledgeStore, MemoError> {
        match mode {
            InitMode::New { path, template, git_remote } => {
                Self::init_new(&path, template.as_deref(), git_remote.as_deref())
            }
            InitMode::Clone { remote_url, local_path } => {
                Self::init_clone(&remote_url, &local_path)
            }
            InitMode::Open { path } => {
                Self::init_open(&path)
            }
        }
    }

    /// 路径 A：从零创建
    fn init_new(path: &Path, template: Option<&str>, git_remote: Option<&str>)
        -> Result<KnowledgeStore, MemoError>
    {
        // 1. 创建目录
        // 2. 创建 .memoforge/config.yaml（空或从模板）
        // 3. 创建 .memoforge/.gitignore（排除运行时文件）
        // 4. 如有模板 → init_from_template()
        // 5. git init
        // 6. 如有 git_remote → git remote add origin <url>
        // 7. 首次 commit: "Initial knowledge base"
        // 8. KnowledgeStore::open(path)
    }

    /// 路径 B：Clone Git 仓库
    fn init_clone(remote_url: &str, local_path: &Path)
        -> Result<KnowledgeStore, MemoError>
    {
        // 1. git2::Repository::clone(remote_url, local_path)
        // 2. 校验 .memoforge/config.yaml 是否存在
        //    - 存在 → 合法的 MemoForge 知识库
        //    - 不存在 → 创建默认 config.yaml + .gitignore（视为导入场景）
        // 3. KnowledgeStore::open(local_path)
        //
        // 错误处理：
        // - GIT_AUTH_FAILED: 认证失败（提示配置 SSH key 或 token）
        // - GIT_REMOTE_UNREACHABLE: 网络不可达
        // - INVALID_PATH: local_path 已存在且非空
    }

    /// 路径 C：打开已有目录
    fn init_open(path: &Path)
        -> Result<KnowledgeStore, MemoError>
    {
        // 1. 校验目录是否存在
        // 2. 如有 .memoforge/config.yaml → 直接 KnowledgeStore::open()
        // 3. 如无 → 创建 .memoforge/ 目录 + 默认 config.yaml + .gitignore
        //    提示用户可通过 GUI 注册分类
        // 4. 如非 Git 仓库 → 提示（不强制 git init）
        // 5. KnowledgeStore::open(path)
    }
}
```

### 6.6 知识图谱可视化

> 对应 PRD §9.4 Nice to Have

#### 6.6.1 数据结构

```rust
/// 图谱节点
pub struct GraphNode {
    pub id: String,           // 知识 ID（相对路径）
    pub title: String,        // 知识标题
    pub category_id: Option<String>,  // 分类 ID
    pub tags: Vec<String>,    // 标签列表
}

/// 图谱边
pub struct GraphEdge {
    pub source: String,       // 源节点 ID
    pub target: String,       // 目标节点 ID
    pub relation: GraphRelationType,  // 关系类型
}

/// 关系类型
pub enum GraphRelationType {
    WikiLink,    // [[wiki-link]] 语法链接
    SharedTag,   // 共享标签
}

/// 知识图谱
pub struct KnowledgeGraph {
    pub nodes: Vec<GraphNode>,
    pub edges: Vec<GraphEdge>,
}
```

#### 6.6.2 构建算法

**性能优化策略：**

1. **WikiLink 边**：遍历每个知识的 markdown 内容，解析 `[[link]]` 语法，2. **SharedTag 边**：使用倒排索引优化，   - 构建 `tag -> Vec<knowledge_id>` 映射
   - 按共享标签数量排序，优先添加共享多个标签的边
   - 默认限制最多 500 条 SharedTag 边

```rust
/// 图谱构建选项
pub struct GraphOptions {
    pub max_nodes: Option<usize>,           // 最大节点数限制
    pub max_edges: Option<usize>,           // 最大边数限制
    pub include_shared_tags: bool,          // 是否包含共享标签边
    pub max_shared_tag_edges: Option<usize>, // 共享标签边最大数量
}

/// 构建知识图谱（带选项）
pub fn build_knowledge_graph_with_options(
    kb_path: &Path,
    options: GraphOptions
) -> Result<KnowledgeGraph, MemoError>;
```

#### 6.6.5 性能考虑

| 知识数量 | 预期节点 | 预期边 | 性能建议 |
|---------|---------|--------|---------|
| < 100 | < 100 | < 500 | 无需优化 |
| 100-500 | < 500 | < 2000 | 默认配置 |
| 500-2000 | < 2000 | < 5000 | 建议设置 max_edges |
| > 2000 | 建议采样 | 建议采样 | 必须设置 max_nodes |
